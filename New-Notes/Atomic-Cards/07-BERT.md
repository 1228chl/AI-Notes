## 📌 一句话严谨定义

BERT（Bidirectional Encoder Representations from Transformers）是一种基于Transformer编码器架构的预训练语言模型，其核心创新在于通过掩码语言模型（Masked Language Model, MLM）和下一句预测（Next Sentence Prediction, NSP）两个自监督预训练任务，在大规模无标注语料上学习深层双向上下文表示，然后通过微调（Fine-tuning）适配下游NLP任务（如文本分类、命名实体识别、问答等），在11项NLP基准任务上刷新了当时的世界最佳成绩。

## 🧠 核心机制与底层原理（深度部分）

BERT的底层原理可以从**预训练任务设计**、**双向上下文建模**和**微调范式**三个维度进行深度拆解：

**预训练任务设计：MLM与NSP**。BERT的预训练包含两个任务：(1) **掩码语言模型（MLM）**：随机遮蔽输入序列中15%的token，让模型预测被遮蔽的token。具体策略是：被选中的token中，80%替换为[MASK]，10%替换为随机token，10%保持不变。这种策略的动机是：如果总是替换为[MASK]，模型可能只学会了处理[MASK] token，而不习惯处理正常token。MLM的损失函数为：$\mathcal{L}_{MLM} = -\sum_{i \in \mathcal{M}} \log P(x_i | \mathbf{x}_{\setminus \mathcal{M}})$，其中 $\mathcal{M}$ 是被遮蔽的位置集合。(2) **下一句预测（NSP）**：判断两个句子是否为连续上下文（二分类任务）。正样本是连续的句子对，负样本是随机组合的句子对。NSP损失为：$\mathcal{L}_{NSP} = -[y \log P(\text{IsNext}) + (1-y) \log P(\text{NotNext})]$。

**双向上下文建模的理论基础**。BERT的核心创新是**双向上下文**：与GPT的单向（从左到右）语言模型不同，BERT通过MLM可以同时利用左右两侧的上下文信息。形式化地，给定被遮蔽位置 $i$ 的上下文 $\mathbf{x}_{\setminus i}$，BERT建模 $P(x_i | \mathbf{x}_{\setminus i})$，而GPT建模 $P(x_i | x_1, ..., x_{i-1})$。双向建模的优势在于：对于"the bank of the river"，BERT可以同时利用"bank"左侧的"the"和右侧的"of the river"来判断"bank"是"河岸"而非"银行"。然而，双向建模也带来了训练-推理不一致的问题：预训练时使用[MASK] token，但微调/推理时输入不包含[MASK]。BERT通过10%随机替换和10%保持不变的策略部分缓解了这一问题。

**微调范式：预训练-微调分离**。BERT采用两阶段范式：(1) **预训练阶段**：在大规模无标注语料（如Wikipedia + BookCorpus，共约33亿词）上训练MLM和NSP任务，学习通用语言表示。(2) **微调阶段**：将预训练的BERT模型与任务特定的输出层组合，在有标注的下游数据集上端到端微调所有参数。微调时，BERT的输入格式为：`[CLS] + 句子A + [SEP] + 句子B + [SEP]`，[CLS] token的输出用于分类任务，每个token的输出用于序列标注任务，句子对的表示用于匹配任务。微调的学习率通常为2e-5到5e-5，训练2-4个epoch即可达到良好性能。

## 🎯 适用场景与边界条件（防错部分）

### 适用场景（什么时候用它）

1. **文本分类任务**：情感分析、新闻分类、垃圾邮件检测等。BERT通过[CLS] token的输出可以有效捕捉整个句子的语义，在小数据集上也能通过微调达到优异性能。例如，在SST-2情感分类数据集上，BERT-Baseline达到93.5%准确率，远超传统方法（如SVM + TF-IDF：85%）。

2. **序列标注任务**：命名实体识别（NER）、词性标注、语义角色标注等。BERT通过每个token的输出可以进行token级别的分类。例如，在CoNLL-2003 NER数据集上，BERT达到92.8%的F1分数。

3. **问答任务**：抽取式问答（如SQuAD数据集）。BERT通过预测答案在上下文中的起始和结束位置来回答问题。例如，在SQuAD 1.1上，BERT达到88.5的F1分数，接近人类水平（89.5）。

### 边界条件（什么时候千万别用它 / 失效条件）

1. **输入序列超过512 token**：BERT的最大序列长度为512 token（包括[CLS]和[SEP]），对于长文档、长文章等超长输入，无法直接使用。解决方案：截断（可能丢失重要信息）、分段编码后聚合、使用长序列模型（如Longformer、BigBird）。

2. **需要实时推理且延迟敏感**：BERT的推理需要计算所有层的自注意力，单条推理在GPU上约需10-50ms，对于高并发、低延迟的在线服务（如搜索排序、实时推荐），可能需要使用模型压缩（如DistilBERT：参数量减少40%，速度提升60%，性能损失仅2%）。

3. **领域差异巨大且无领域预训练**：BERT的预训练数据是通用语料（Wikipedia + BookCorpus），如果目标领域与通用语料差异巨大（如医疗、法律、金融），直接微调可能效果不佳。解决方案：在领域语料上继续预训练（Domain-Adaptive Pre-training），如BioBERT（生物医学）、FinBERT（金融）、LegalBERT（法律）。

## 📊 深度案例拆解（必须包含正例与反例）

### 正例（为什么对？）

**场景**：使用BERT进行医疗文本中的疾病实体识别

**数据**：5000条中文医疗记录，标注了疾病实体（如"高血压"、"糖尿病"）

**正确应用**：
1. **领域预训练**：在中文医疗语料（如PubMed中文摘要、医疗问答）上继续预训练BERT，学习医疗领域知识
2. **微调策略**：在BERT输出上添加CRF层（条件随机场），建模标签之间的转移关系
3. **标签方案**：使用BIO标注（B-疾病、I-疾病、O）
4. **评估指标**：使用实体级别的精确率、召回率、F1（而非token级别）

**训练结果**：
- 通用BERT + CRF：F1 = 82.3%
- 医疗BERT + CRF：F1 = 89.7%（提升7.4%）

**关键决策**：选择在领域语料上继续预训练，因为医疗文本中的疾病实体（如"2型糖尿病"、"急性心肌梗死"）与通用语料中的词汇分布差异大，通用BERT难以捕捉这些领域特定的语义模式。

### 反例（为什么错？）

**场景**：使用BERT处理长文档分类（每篇文档约5000 token）

**错误做法**：
1. 将文档截断到512 token
2. 只使用前512 token进行分类
3. 忽略文档后半部分的内容

**失败原因分析**：
1. **信息丢失**：文档的关键论点可能在后半部分，截断导致模型无法看到完整信息
2. **偏置**：模型可能学到"重要信息在前512 token"的虚假模式，在测试集上表现良好但在真实场景中失败
3. **评估偏差**：测试集也被截断，因此评估结果高估了真实性能

**正确做法**：
- 使用滑动窗口：将文档分为多个512 token的片段，每个片段独立编码，然后通过注意力池化（Attention Pooling）或最大池化聚合
- 使用层次化BERT：先编码句子级（每个句子独立编码），再编码文档级（使用Transformer处理句子序列）
- 使用长序列模型：Longformer（滑动窗口 + 全局注意力）支持最长4096 token

## 💼 面试/实战深挖指南（附加价值）

*   **初级追问**：如果面试官问"请解释一下BERT"，回答的要点有哪些？
    1. 给出核心定义：基于Transformer编码器的双向预训练语言模型
    2. 解释预训练任务：MLM（掩码语言模型）和NSP（下一句预测）
    3. 解释双向上下文：同时利用左右两侧的信息
    4. 解释微调范式：预训练 + 下游任务微调
    5. 列举经典应用：文本分类、NER、问答

*   **高级追问**：如果面试官问"BERT和GPT有什么区别"或"什么时候用BERT，什么时候用GPT"，应该如何结合上述"边界条件"进行分层应答？
    1. **架构差异**：BERT是编码器（双向），GPT是解码器（单向）
    2. **任务差异**：BERT擅长理解任务（分类、NER），GPT擅长生成任务（对话、文本生成）
    3. **预训练目标**：BERT使用MLM（双向），GPT使用CLM（单向）
    4. **选择建议**：理解任务→BERT；生成任务→GPT；需要双向上下文→BERT；需要自回归生成→GPT
    5. **融合趋势**：T5、BART等模型融合了编码器和解码器的优势
