# 一、大模型发展史

- 现代CNN的基本结构：卷积层、池化层和全连接层的交替使用

# 二、大模型开发技术

## 1. 提示词工程（Prompt Engineering）

通过精妙的指令设计，约束模型行为，引导模型进行结构化思考。

**核心价值**：
- 无需修改模型参数
- 即时生效，成本低
- 适合快速迭代

---

## 2. 模型微调（Fine-tuning）

通过数据驱动的方式，从根本上调整模型参数，使其在特定领域内更可靠。

**核心价值**：
- 深度定制模型行为
- 提升特定任务性能
- 需要高质量标注数据

---

## 3. 检索增强生成（RAG）

将大模型与外部知识源（如数据库、文档库、互联网）连接。在回答问题时，先检索相关证据，再让模型基于证据生成答案。

**核心价值**：
- 解决模型的"知识更新"问题
- 解决"私有知识"问题
- 扼杀事实性幻觉的利器

---

## 4. AI Agent

Agent将大模型从"内容生成器"提升为"动作执行者"，其核心架构天然包含了对抗幻觉的机制。

**核心价值**：
- 模型作为"大脑"，可以规划步骤
- 调用工具验证猜想
- 构建动态的、自我修正的系统

---

## 5. 未来趋势

构建"**微调过的模型大脑 + RAG知识心脏 + Agent协作四肢**"的超级个体。

# 三、大模型幻觉问题

## 1. 什么是幻觉

大模型的幻觉是指大模型的生成结果中包含了无根据的或错误的内容。

---

## 2. 幻觉类型

### 2.1 事实性幻觉（Factuality）

**表现**：
- 事实不一致（factual inconsistency）
- 捏造事实（fabrication）

**原因**：
- 知识过时
- 数据不准确

### 2.2 忠实性幻觉（Faithfulness）

**表现**：
- 不遵循指令（Instruction inconsistency）
- 不遵循上下文（in-Context inconsistency）
- 回复内容逻辑不一致（Logical inconsistency）

**原因**：
- 指令的模糊性与复杂性
- 推理能力的不足

---

## 3. 解决方案

| 方案 | 说明 |
|------|------|
| **提示词工程** | 通过清晰的指令减少幻觉 |
| **RAG** | 引入外部知识库验证 |
| **模型微调** | 训练模型生成事实性内容 |
| **AI Agent** | 通过工具调用验证信息 |

# 四、提示词工程

## 1. 提示词工程原则

基于OpenAI官方文档，提炼出5条大原则：

1. 清晰的指令
2. 文本参考
3. 复杂任务拆分简单子任务
4. 给模型"思考"的时间
5. 借助外部工具

---

## 2. 清晰的指令

### 2.1 详细的描述

不要描述得太笼统，尽量多的提供重要的详细信息或上下文。

**示例**：
```
❌ 不好的提示：写一篇文章
✅ 好的提示：请写一篇关于人工智能在医疗领域应用的科普文章，目标读者是高中生，字数约1500字，包含3个实际案例
```

### 2.2 让模型充当某个角色

使用大模型时，可以让模型充当一个角色，这样模型会更专业更明确地回复。

**示例**：
```
你是一位资深的Python开发专家，请帮我分析以下代码的问题...
```

### 2.3 使用分隔符标明输入的不同部分

中括号、XML标签、三引号等分隔符可以帮助划分要区别对待的文本。

**示例**：
```
请将以下文本翻译成英文：
"""
人工智能正在改变世界。
"""
```

### 2.4 提供示例

大模型仅依赖几个示例样本就可以学习新任务。

**示例**：
```
请将以下句子分类为正面或负面情感：

示例1：
输入："这部电影太棒了！"
输出：正面

示例2：
输入："服务态度很差"
输出：负面

请分类：
输入："今天天气真好"
输出：
```

### 2.5 指定输出长度

要求模型生成给定目标长度的输出。

**示例**：
```
请用100字以内总结这篇文章的核心观点
```

---

## 3. 文本参考

基于文本文档，辅助大模型问答，降低模型"幻觉"问题。

**示例**：
```
根据以下文档回答问题：
"""
[文档内容]
"""
问题：xxx
```

---

## 4. 复杂任务拆分为简单子任务

将复杂任务分解为多个简单步骤。

**示例**：
```
请按以下步骤分析这篇论文：
1. 首先，总结论文的研究背景
2. 然后，提取论文的主要贡献
3. 接着，分析论文的实验结果
4. 最后，给出你对论文的评价
```

---

## 5. 给模型"思考"的时间

### 5.1 思维链提问

在例子中展示一步步的推理过程，模型会模仿这种"慢思考"。

**示例**：
```
请一步一步思考：

问题：一个农场有15只鸡和8只鸭，如果卖掉3只鸡，剩下多少只家禽？

思考过程：
1. 原来有鸡：15只
2. 原来有鸭：8只
3. 卖掉鸡：3只
4. 剩下的鸡：15 - 3 = 12只
5. 剩下的家禽总数：12 + 8 = 20只

答案：20只
```

### 5.2 让大模型显式写出推理过程

不是直接给出结果，而是显式写出推理过程。

---

## 6. 借助外部工具

### 6.1 联网搜索工具

解决模型不知道实时信息的问题。

### 6.2 代码执行工具

解决需要精确计算或数据处理时，模型自身"算得不准"的问题。

### 6.3 数据库/知识库工具

访问私有数据和知识库。

### 6.4 外部API调用

解决专业需求，比如天气查询、航班查询、地图导航。

# 五、RAG检索增强生成

## 1. 什么是RAG

RAG（Retrieval-Augmented Generation）是一种将检索与生成相结合的技术，先从外部知识库中检索相关信息，再让模型基于检索结果生成答案。

**核心流程**：
```
用户问题 → 检索相关文档 → 将文档+问题送入LLM → 生成答案
```

---

## 2. RAG的优势

| 优势 | 说明 |
|------|------|
| **减少幻觉** | 基于检索到的事实生成答案 |
| **知识更新** | 可以随时更新知识库 |
| **私有知识** | 可以接入企业内部文档 |
| **可追溯** | 可以给出答案的来源 |

---

## 3. RAG核心组件

### 3.1 文档处理

- 文档加载
- 文本分块（Chunking）
- 向量化（Embedding）

### 3.2 向量数据库

存储文档的向量表示，支持快速相似度搜索。

常用向量数据库：
- FAISS
- Chroma
- Pinecone
- Milvus

### 3.3 检索策略

- **稠密检索**：基于向量相似度（余弦相似度、内积）
- **稀疏检索**：基于关键词匹配（BM25）
- **混合检索**：结合稠密和稀疏检索

### 3.4 生成模型

使用LLM基于检索结果生成答案。

---

## 4. RAG代码示例

```python
from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import FAISS
from langchain.llms import OpenAI
from langchain.chains import RetrievalQA

# 1. 加载文档
loader = TextLoader('document.txt')
documents = loader.load()

# 2. 文本分块
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = text_splitter.split_documents(documents)

# 3. 创建向量数据库
embeddings = OpenAIEmbeddings()
vectorstore = FAISS.from_documents(chunks, embeddings)

# 4. 创建检索问答链
llm = OpenAI()
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=vectorstore.as_retriever()
)

# 5. 提问
answer = qa_chain.run("什么是人工智能？")
print(answer)
```

# 六、AI Agent

## 1. 什么是AI Agent

AI Agent是以大语言模型为驱动，具备自主理解、感知、决策和执行能力的智能体。

**核心能力**：
- **感知**：理解用户输入和环境信息
- **规划**：制定完成任务的步骤
- **执行**：调用工具完成具体操作
- **反思**：评估结果并调整策略

---

## 2. Agent核心架构

```
用户输入 → LLM（大脑） → 规划 → 工具调用 → 执行 → 结果 → 反馈
```

**关键组件**：
- **LLM**：作为大脑，负责理解、推理和决策
- **工具（Tools）**：外部API、代码执行器、搜索引擎等
- **记忆（Memory）**：短期记忆（上下文）和长期记忆（向量数据库）
- **规划（Planning）**：任务分解、策略制定

---

## 3. Agent实现框架

### 3.1 LangChain Agent

```python
from langchain.agents import initialize_agent, Tool
from langchain.llms import OpenAI

# 定义工具
def search(query):
    # 实现搜索逻辑
    return "搜索结果"

tools = [
    Tool(
        name="Search",
        func=search,
        description="用于搜索互联网获取最新信息"
    )
]

# 创建Agent
llm = OpenAI(temperature=0)
agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent="zero-shot-react-description"
)

# 运行
result = agent.run("今天北京的天气怎么样？")
print(result)
```

### 3.2 AutoGPT

AutoGPT是一个能够自主执行任务的AI Agent框架。

**特点**：
- 自主设定目标
- 自主规划步骤
- 自主执行任务
- 自主反思和调整

---

## 4. Agent应用场景

| 场景 | 说明 |
|------|------|
| **智能客服** | 理解用户问题，调用知识库，生成回复 |
| **数据分析** | 执行SQL查询，生成可视化报告 |
| **代码开发** | 编写、调试、测试代码 |
| **内容创作** | 撰写文章、生成图片、制作视频 |
| **任务自动化** | 自动化处理重复性工作 |

---

## 5. Agent挑战

| 挑战 | 说明 |
|------|------|
| **安全性** | Agent可能执行危险操作 |
| **可控性** | 难以完全控制Agent行为 |
| **成本** | 多次调用LLM成本较高 |
| **延迟** | 多步推理可能导致响应变慢 |
