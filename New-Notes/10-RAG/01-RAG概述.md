# RAG概述

**标签：** #RAG #LLM

---

## 1. 什么是RAG

RAG（Retrieval-Augmented Generation）是一种将检索与生成相结合的技术，先从外部知识库中检索相关信息，再让模型基于检索结果生成答案。

**核心流程**：
```
用户问题 → 检索相关文档 → 将文档+问题送入LLM → 生成答案
```

---

## 2. RAG的优势

| 优势 | 说明 |
|------|------|
| **减少幻觉** | 基于检索到的事实生成答案 |
| **知识更新** | 可以随时更新知识库 |
| **私有知识** | 可以接入企业内部文档 |
| **可追溯** | 可以给出答案的来源 |

---

## 3. RAG核心组件

### 3.1 文档处理

- 文档加载
- 文本分块（Chunking）
- 向量化（Embedding）

### 3.2 向量数据库

存储文档的向量表示，支持快速相似度搜索。

常用向量数据库：
- FAISS
- Chroma
- Pinecone
- Milvus

### 3.3 检索策略

- **稠密检索**：基于向量相似度（余弦相似度、内积）
- **稀疏检索**：基于关键词匹配（BM25）
- **混合检索**：结合稠密和稀疏检索

### 3.4 生成模型

使用LLM基于检索结果生成答案。
